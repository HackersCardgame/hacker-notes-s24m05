Been watching quite a few talks from AV pros on what they've been working with... was looking into this for my local hackerspace awhile back for some smaller scale talks. Here's some notes/ramblings.

A lot depends on what the type and scale of event is. I assume you're talking about "A virtual tech conference with multiple presenters, etc."

Twitch/Youtube is generally a good destination for streams. You can self-host a stream too, that's a sliding scale of "how much time you want to spend" versus "how much money you want to spend"

A lot of people self-host an intermediate box or use something like https://restream.io/ that can show stuff while switching feeds if you're having multiple feeds from different presenters that switch throughout the day.

If you're hosting a panel with multiple people and want to broadcast to a wider group, Zoom is a pretty good option. You can invite the panelists to the call itself and then stream the call as a whole to Twitch/YouTube directly from Zoom's backend. Watchers never have to touch Zoom itself.

Zoom also has a "webinar" feature that allows you to bring in people into the platform itself, which has the advantage of having features like Q&A and chat built in, but it's quite expensive and requires people to join the Zoom webinar itself. Can still be streamed too.

If it's just talks done by single participants, the two major ways I've seen it done is have the participant stream (using something like OBS) or using any of the many calling platforms out there (Skype, Jitsi, etc.).

Big Blue Button is an open source video conferencing system, mostly for education. No real broadcasting but a lot of good tools for a class or lab. Built-in whiteboards, etc.

For a stream, if you want to take text comments that the panelists are going to answer on stream, there's a handful of solutions based on Twitch chat and Twitter (using a Hashtag or @ account). This allows a moderator to filter out the spam and only present the good stuff to the panel hosts or participants.
