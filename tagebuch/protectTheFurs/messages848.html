Yeah, one easy example.

Many basic reverse engineering programs can pull out strings from compiled code, or will use a set of regex patterns to attempt to extract interesting data from source code.

If you are able to break up the actual public key in a way that disrupts the patterns it is looking for you can hide the actual key.

If you then put a fake key reading routine into the code that isn’t utilized by the actual encryption, this could cause an adversary to find this fake key early on.

As a blue team, you could then detect this key was used, or just that a valid key was not used.

There are a lot of variations on this idea, but it helps to slow down the process and potentially provide warning signs to internal security.
